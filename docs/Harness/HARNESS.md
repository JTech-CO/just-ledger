# just-ledger — Claude Code 작업 하네스 (Harness)

**버전**: 0.1
**작성일**: 2026년 07월 20일
**관계 문서**: 루트 `CLAUDE.md`(전역 규칙·불변식), `PROGRESS.md`(상태 인계), `docs/TECH-WHITEPAPER.md`, `docs/DESIGN-WHITEPAPER.md`, `contracts/*.schema.json`

> 이 문서는 **"무엇을 만드는가"(백서)가 아니라 "어떻게 진행·검증·복구하는가"(운영 규율)** 를 정의한다. 각 phase의 "완료"는 코드가 도는 것이 아니라 **측정 가능한 게이트(DoD) 통과**다. 11개 언어가 얽힌 시스템에서는 "에러 없이 도는데 기능은 사실상 안 됨"이 특히 흔하므로, 게이트로 막지 않으면 잘못된 완료 판정이 다음 phase로 전파된다.

---

## 0. 사용법

### 0.1 세션 루프

1. **시작**: `PROGRESS.md` 읽기 → 현재 phase·다음 할 일·미결 질문 확인 → 본 문서의 해당 phase 절 + 관련 백서 절 확인.
2. **작업**: 한 번에 한 phase, **한 번에 한 언어**. 각 작업 단위마다 그 phase의 검증 명령을 돌려 게이트로 확인.
3. **종료**: `PROGRESS.md` 갱신(완료 체크·다음 할 일·새 미결 질문·결정 로그) → 커밋.

### 0.2 phase 완료 판정

- DoD 항목이 **전부** 충족되어야 완료. 하나라도 미달이면 미완료로 두고 다음 phase로 가지 않는다.
- 게이트를 못 넘는데 우회 충동이 들면 → §3 멈춤 규칙.

### 0.3 의존 순서

`M0 → M1 → M2 → M3 → M4 → M5 → M6 / M7 → M8 → M9`

M6(분석)과 M7(실시간)은 병렬 가능하지만 기본은 위 순서를 따른다. M7은 M1·M2 게이트만 통과하면 착수 가능하다.

### 0.4 `PROGRESS.md` 최소 구성

현재 phase / 직전에 끝낸 것 / 다음 할 일 / 미결 질문 / 결정 로그(날짜·결정·이유). 세션이 끊겨도 **이 파일만 읽으면 이어서 작업 가능**해야 한다.

폴리글랏 프로젝트 특성상 다음 두 항목을 추가로 유지한다.

- **툴체인 상태**: 각 언어의 확인된 버전과 마지막 검증 일자
- **계약 변경 로그**: `contracts/` 또는 copybook이 바뀌면 영향받는 언어 모듈을 함께 기록

### 0.5 언어별 작업 규율

- 한 세션에서 두 언어를 동시에 건드리지 않는다. COBOL 작업 중 Haskell을 손대면 양쪽 다 품질이 떨어진다.
- 각 `modules/*`, `services/*` 디렉터리에 자체 `CLAUDE.md`를 두고 빌드·테스트·실행 명령과 언어 고유 컨벤션을 명시한다. 루트 `CLAUDE.md`에 11개 언어 규칙을 전부 적재하지 않는다.
- 새 모듈은 **계약(스키마 또는 copybook)과 골든 픽스처를 먼저** 작성한 뒤 구현에 들어간다.

## 1. Phase별 진입조건 · 할 일 · DoD · 검증

### M0 — 툴체인 및 스캐폴딩 ★

- **진입조건**: 없음 (프로젝트 시작점)
- **할 일**: 레포 초기화 → 개발 환경 고정(Nix flake 또는 devcontainer) → `Makefile` 언어별 타깃 → `.gitattributes` 초안 → 루트 `CLAUDE.md` → `PROGRESS.md` → `contracts/` 골격 → CI 경로 매트릭스
- **참조**: 기술 백서 §3.3, §6, §8.2
- **DoD**:
  1. `make toolchain-check` 가 11개 툴체인(node, go, rustc/wasm-pack, ghc/cabal, cobc, swipl, Rscript, julia, elixir/mix, lua, psql) 버전을 출력하고 전부 성공 종료.
  2. 클린 클론 상태에서 **문서화된 단일 명령**으로 전체 빌드가 성공한다.
  3. `make lint`, `make build` 그린.
  4. CI가 변경 경로 기반 매트릭스로 언어별 잡을 분리 실행하고, 무관한 언어 잡은 스킵된다.
- **검증**: `make toolchain-check` / `docker compose build` / CI 그린
- **주의**: 여기서 아낀 시간은 이후 전 단계에서 10배로 돌아온다. **로컬에서만 되고 CI에서 안 되는 상태로 M1에 진입하지 않는다.** 툴체인 버전은 이 시점에 고정하고 `PROGRESS.md`에 기록한다.

### M1 — 데이터 및 영속화 (PostgreSQL / PLpgSQL) ★

- **진입조건**: M0 게이트 통과
- **할 일**: `db/migrations` → 도메인 테이블 → `CHECK` 제약 → 이중분개 트리거 → 잔액 롤업 함수 → 기간 집계 뷰 → RLS 정책 → 변경 시 `NOTIFY` 발행
- **참조**: 기술 백서 §2.3
- **DoD**:
  1. 마이그레이션 up/down 왕복 성공, 스키마 diff 0.
  2. 무작위 100,000 txn 삽입 후 **INV-1 위반 0건**.
  3. 불균형 txn 삽입 시도가 **반드시 실패**한다 (음성 테스트 통과).
  4. `settled` 기간의 UPDATE/DELETE 시도가 전부 거절된다 (INV-3).
  5. 잔액 롤업 함수 결과가 원장 전수 합산 결과와 전 계정·전 기간 일치.
  6. `NOTIFY` 페이로드가 계약 스키마를 준수한다.
- **불변식**: INV-1 / INV-2 / INV-3 / INV-5. **1건이라도 위반 시 통과 금지.**
- **검증**: `make test-db`
- **주의**: 균형 검사 트리거를 `BEFORE`로 두면 다중 entry 삽입 중간 상태에서 오탐한다. **deferrable constraint trigger**로 트랜잭션 커밋 시점에 검사한다.

### M2 — API 및 서버 골격 (JavaScript)

- **진입조건**: M1 게이트 통과
- **할 일**: `contracts/` 스키마 확정 → JSDoc `@typedef` 생성 → Fastify 라우트 → 어댑터 인터페이스 정의(worker/prolog/realtime, 이 시점엔 스텁) → React 셸 → `LedgerTable` 최소판
- **참조**: 기술 백서 §2.2, §3.1, §4.1
- **DoD**:
  1. `GET /health` 200, 기동 후 3초 이내.
  2. 핵심 엔드포인트(계정 CRUD, txn 조회·생성, 기간 잔액) 왕복 테스트 통과.
  3. 요청/응답이 `contracts/*.schema.json` 검증을 통과하고, 스키마 위반 요청은 400으로 거절된다.
  4. 브라우저에서 수기 거래 입력 → 저장 → 재조회 시 금액이 **문자열 왕복 무손실**.
  5. **금액이 `Number`를 경유하는 코드 경로 0개** (정적 검사 통과, INV-4).
- **검증**: `make test-api` / `make check-no-float`
- **주의**: 어댑터는 스텁이라도 **실제 인터페이스 시그니처를 확정**하고 넘어간다. 이후 언어 모듈이 이 계약에 맞춰 붙는다.

### M3 — 인제스트 파이프라인 (Rust WASM + Go) ★

- **진입조건**: M2 게이트 통과
- **할 일**: Rust 파서(CSV/OFX/QIF) → 정규화 → BLAKE3 지문 → Argon2id 키 파생 및 암호화 → `wasm-pack` 빌드 → Web Worker 연결 → Go 워커 큐·스케줄러 → draft txn 생성 → 환율 폴링
- **참조**: 기술 백서 §4.2-2, §4.3(중복 제거)
- **DoD**:
  1. 골든 픽스처 은행 CSV **3종**에 대해 파싱 결과가 기대 출력과 **완전 일치**.
  2. 동일 파일 재업로드 시 신규 txn **0건**.
  3. 50MB CSV 파싱 3초 이내, 메인 스레드 롱태스크 0건.
  4. 서버가 수신한 페이로드에 평문 상대처·적요가 존재하지 않는다 (바이트 검사).
  5. 워커 강제 종료 후 재시작 시 미완료 배치가 자동 재개된다.
- **불변식**: 서버 로그·DB·임시 파일 어디에도 명세서 원문이 평문으로 남는 경로 0개.
- **검증**: `make test-ingest` / `make bench-wasm`
- **주의**: 인코딩(CP949, UTF-8 BOM), 날짜 형식, 천단위 구분자, 음수 표기(`△`, 괄호)는 은행마다 다르다. 픽스처에 **실제 3사 포맷**을 넣는다. 하나만 맞추면 나머지에서 전부 깨진다.

### M4 — 추론 계층 (Prolog + Lua)

- **진입조건**: M3 게이트 통과
- **할 일**: Prolog 사실 적재 인터페이스 → 분류 규칙 → 이체 페어 매칭 → 정기결제 탐지 → HTTP 서비스화 → Go 워커에 Lua 샌드박스 임베드 → 사용자 규칙 실행 경로
- **참조**: 기술 백서 §4.3(이체 페어 매칭, 정기 결제, Lua 샌드박스)
- **DoD**:
  1. 라벨링된 골든 세트 500건에서 분류 정확도 ≥ **0.90**.
  2. **이체 페어 매칭 거짓양성 0건.**
  3. 1,000건 분류 1초 이내.
  4. Lua 스크립트가 `os`·`io`·`package`·`debug`·`load` 접근을 시도하면 전부 차단된다 (음성 테스트 5종).
  5. 무한 루프 Lua 스크립트가 100ms 내에 강제 중단된다.
  6. 분류 결과에 **근거 규칙명이 함께 반환**된다 (Inspector에 표시하기 위함).
- **불변식**: 이체 페어 거짓양성 1건이라도 발생 시 통과 금지. 정확도를 올리려고 금액 근사 매칭을 도입하지 않는다.
- **검증**: `make test-classify` / `make test-sandbox`

### M5 — 정산 계층 (COBOL + Haskell) ★

- **진입조건**: M4 게이트 통과
- **할 일**: copybook 정의 → Go 고정폭 레코드 생성기 → COBOL 마감 배치(정산·이자·상각·반올림·리포트) → JavaScript 참조 구현(`decimal.js`) → 대조 하네스 → Haskell DSL 렉서·파서·타입검사·평가기
- **참조**: 기술 백서 §4.3(은행가 반올림, 규칙 DSL, 정합성 대조)
- **DoD**:
  1. **10,000 txn 마감 결과가 JavaScript 참조 구현과 전 항목 일치하고 차이가 0원이다.** ← 프로젝트 전체에서 가장 중요한 게이트
  2. 상각 스케줄의 마지막 회차 종료 잔액이 정확히 0.
  3. 0.5 경계 반올림 케이스 전수 테스트에서 NEAREST-EVEN 준수.
  4. copybook 레코드 길이를 초과하는 입력이 **COBOL 실행 전에** 거절된다.
  5. DSL: 유효 규칙 20종 파싱 성공, 오류 규칙 20종이 **소스 위치 정보와 함께** 실패한다.
  6. 마감 10,000 txn 2초 이내.
- **불변식**: 정합성 불일치가 1건이라도 있으면 마감을 커밋하지 않는다. **반올림 규칙이나 허용 오차를 조정해 통과시키지 않는다.**
- **검증**: `make test-settlement` / `make test-dsl` / `make parity-settlement`
- **주의**: `COMP-3` 부호 니블 표현은 플랫폼·컴파일 옵션에 따라 달라진다. 컨테이너 이미지와 `cobc` 플래그를 고정하고 골든 픽스처로 **바이트 단위** 회귀를 잡는다.

### M6 — 분석 계층 (R + Julia)

- **진입조건**: M5 게이트 통과
- **할 일**: R STL 분해 → MAD 이상치 탐지 → 부트스트랩 초과 확률 → ggplot 테마 토큰화 → SVG 산출(라이트/다크 2벌) → Julia 몬테카를로 → 상환 순서 DP → 산출물 캐시
- **참조**: 기술 백서 §4.3(이상치 탐지, 현금흐름 시뮬레이션), 디자인 백서 §4.3(`ReportChart`)
- **DoD**:
  1. 고정 시드에서 R·Julia 산출물이 재실행 시 **바이트 단위 동일** (결정론).
  2. 알려진 이상치를 주입한 데이터셋에서 재현율 ≥ 0.95, 거짓양성률 ≤ 0.05.
  3. 몬테카를로 10,000 경로에서 `NaN`·발산 0건, 95% 신뢰구간 폭이 기준 이하.
  4. 생성 SVG가 라이트·다크 두 벌 모두 산출되고 대비비 AA를 충족한다.
  5. 입력 해시가 동일하면 재실행 없이 캐시가 반환된다.
- **검증**: `make test-analytics` / `make test-simulation`
- **주의**: Julia는 첫 호출 JIT 지연이 크다. 워커 기동 시 더미 호출로 예열하고, 벤치마크는 예열 이후 측정한다.

### M7 — 실시간 계층 (Elixir)

- **진입조건**: M1·M2 게이트 통과 (M3~M6과 병렬 가능)
- **할 일**: Phoenix 앱 → PostgreSQL `LISTEN` 구독 → 채널 → 예산 임계 감시 GenServer → 알림 감독 트리 → 클라이언트 연결 및 store 병합
- **참조**: 기술 백서 §4.1, §4.2-2
- **DoD**:
  1. DB 변경 → 브라우저 수신 **p95 300ms 이내**.
  2. 동시 연결 100개 유지 상태에서 메시지 유실 0건.
  3. Elixir 프로세스를 강제 종료해도 자동 복구되고, 재접속 시 미수신 이벤트가 보정된다.
  4. 동일 예산 임계에 대한 알림이 중복 발송되지 않는다.
  5. PostgreSQL 커넥션 풀 총합이 `max_connections` 를 초과하지 않는다.
- **검증**: `make test-realtime`

  1. `github-linguist --json` 기준 메인 제외 **전 언어 ≥ 5.0%**.
  2. 계측 대상 11개 외 언어가 통계에 등장 **0건**.
  3. 메인(JavaScript) ≤ 35%.
  4. 위 3항을 의도적으로 위반시킨 브랜치에서 **CI가 실패**함을 확인 (음성 테스트).
  5. 클린 환경에서 `docker compose up` → 전체 스택 기동 → 스모크 시나리오 통과.
  6. README에 11개 언어별 선정 근거가 각 1문장 이상 기재되어 있다.
- **주의**: 비율을 맞추려고 **주석 패딩·무의미한 코드를 넣지 않는다.** 부족하면 실제 기능 또는 골든 테스트를 추가한다. 근거를 한 문장으로 쓸 수 없는 언어는 프로젝트에서 뺀다.

## 3. 멈춤 규칙 (STOP)

### 3.1 멈춰야 하는 상황

- 같은 에러/테스트 실패를 **서로 다른 방법으로 3회** 시도해도 미해결.
- DoD 게이트(특히 INV-1~5, M5 정합성, M4 거짓양성 0)를 못 넘는데 우회 충동.
- 불변식(루트 `CLAUDE.md`)을 깨야만 통과 가능.
- 큰 아키텍처 변경 또는 **언어 교체**가 필요해 보임.
- 외부 제약(툴체인 미지원, 대상 접근 불가, 정책 위반 소지).

### 3.2 멈출 때 절차

1. `PROGRESS.md`에 기록: **증상 / 재현 방법 / 시도한 것들 / 가설 / 막힌 지점**.
2. 사용자에게 위 요약을 보고하고, 선택지가 있으면 제시한 뒤 결정을 요청한다.
3. 결정 전까지 **불변식을 깨는 임시 우회를 만들지 않는다.**

### 3.3 절대 금지

- 테스트 삭제·약화 또는 게이트 수치 임의 하향으로 "통과" 위장.
- **정합성 허용 오차 도입** — 마감 결과 차이는 0원이어야 하며, "1원 이내 허용"으로 완화하지 않는다.
- 이체 페어 매칭에 금액 근사 매칭 도입.
- 금액을 부동소수점으로 처리하는 임시 우회.
- 정합성·불변식이 깨진 채 다음 phase 진행.
- 아키텍처·패키지 경계 임의 변경, 사용자 승인 없는 **언어 교체 또는 추가**.
- Linguist 비율을 맞추기 위한 주석 패딩·무의미한 코드 생성.
- 시크릿(`.env`)·대용량 산출물·의존성 디렉터리(`deps/`, `target/`, `_build/`) 커밋.

## 4. 검증 우선순위 (한 줄 요약)

**불변식(INV-1~5) > 정합성(COBOL↔참조 구현) > 핵심 기능 실효(분류·인제스트) > 통합(실시간·분석) > UX/배포/언어 비율.**

앞 단계의 게이트가 깨지면 뒤 단계 작업은 의미가 없으므로 항상 앞에서부터 굳힌다. **언어 구성 비율은 가장 마지막 관심사다** — 원장이 틀리는 가계부는 언어가 몇 개든 가치가 없다.
